            #### component.toml

            The [component.toml](./../../../start-here/configuration/componenttoml/) file, defines a component, and one or more components make up an application in Heim.

            <br/>

            `component.toml`
            ```toml
            name = "my-csharp-application"
            version = "0.1.0"

            [registry.metadata]
            authors = [""]
            description = "Description of my-csharp-application"
            license = ""
            categories = [""]

            [build.dev]
            build = "dotnet build"
            wasm_path = "./bin/Debug/net10.0/wasi-wasm/native/my-csharp-application.wasm"

            [build.prod]
            build = "dotnet build . --configuration Release"
            wasm_path = "./bin/Release/net10.0/wasi-wasm/publish/my-csharp-application.wasm"
            ```
            _a simple component.toml, with explanatory comments_

            #### application.toml

            The [application.toml](./../../../start-here/configuration/applicationtoml/) is what defines an application. A application.toml is required for deploying an application to Heim.

            ```toml
            name = "my-csharp-application"
            component_target = 'my-csharp-application:0.1.0'

            method = 'GET'
            ```

            Open `Program.cs` and take a look at the code we have:

            `Program.cs`
            ```csharp
            using System.Text;
            using static Nor2WorldWorld.wit.imports.wasi.http.v0_2_1.ITypes;

            namespace Nor2WorldWorld.wit.exports.wasi.http.v0_2_1;

            public class IncomingHandlerImpl : IIncomingHandler
            {
                public static void Handle(IncomingRequest request, ResponseOutparam responseOut)
                {
                    var content = Encoding.ASCII.GetBytes("Hello, World!");
                    var headers = new List<(string, byte[])> {
                        ("content-type", Encoding.ASCII.GetBytes("text/plain")),
                        ("content-length", Encoding.ASCII.GetBytes(content.Length.ToString()))
                    };

                    SendResponse(responseOut, headers, content);
                }

                public static void SendResponse(ResponseOutparam responseOut, List<(string, byte[])> headers, byte[] bodyBytes)
                {
                    var response = new OutgoingResponse(Fields.FromList(headers));
                    var body = response.Body();
                    // This handling is neeeded, because of an bug in Wit-bindgen for dotnet.
                    // https://github.com/bytecodealliance/wit-bindgen/pull/1215
                    var responseHeaders = Fields.FromList(new List<(string, byte[])>()).AsOk;
                    try {
                        responseHeaders = Fields.FromList(headers).AsOk;
                    } catch(Exception) {}

                    var response = new OutgoingResponse(responseHeaders);
                    response.SetStatusCode(200);
                    var body = response.Body().AsOk;

                    ResponseOutparam.Set(responseOut, Result<OutgoingResponse, ErrorCode>.Ok(response));
                    WriteToOutgoingBody(body, bodyBytes);
                    using (var stream = body.Write().AsOk) {
                        stream.BlockingWriteAndFlush(bodyBytes);
                    }

                    OutgoingBody.Finish(body, null);
                }
            }
            ```

## Summary

<br />

1. - run `heim new`
   - select the csharp-http `template`
   - choose `HTTP method`
   - choose trigger `path`
   - set application `name`

   {" "}

   <br />
   <br />

2. - open the created application folder/project in your text editor

   _Take note of the configuration files:_

   :::note[component.toml]
   component.toml defines a component.

   An application is made up by one or more components.
   :::

   :::note[application.toml]
   application.toml defines an application.
   :::

   <br />

   <br />

3. - open `Program.cs`:

   :::note[Program.cs]
   program.cs contains boilerplate source code for your application.
   :::

   {" "}

   <br />

   - run `heim start` to start the Heim runtime

   :::note[Heim portal]
   The Heim portal comes included with a Heim installation.

   Once the Heim runtime is started, the Heim portal is available on [http://127.0.0.1:3000/heim/portal/](http://127.0.0.1:3000/heim/portal/).
   :::

   {" "}

   <br />

   - run `heim deploy` to build and deploy the application.

   :::note[heim deploy --release]
   The `heim deploy --release` creates a _release_ build.

   Just type `heim deploy` to create a _development_ build.
   :::

   {" "}

   <br />

   - run `heim deploy --cloud` to deploy the application to the cloud instance.

   {" "}

   <br />

   :::note[access application]
   The application is accessible on:

   `http://127.0.0.1:3000/${application_path}`
   :::

   :::tip[heim runtime]
   The Heim runtime will print out relevant information about the application.
   :::

   <br />

   <br />

4. - edit the `Cargo.toml` file to add useful dependencies
   - edit the `component.toml` file to add _environment variables_
   - edit the `application.toml` file to change the _HTTP method_
   - edit the `application.toml` file to change the _trigger path_
   - edit the `application.toml` file to set environment variable values
   - edit the `program.cs` file to use new environment variables
   - edit the `program.cs` file to set up return and accept values
   - _deploy_ the application again to see your changes

   {" "}

   <br />

   {" "}

   <br />

5. - use the `Ctrl+C` shortcut to shut down the Heim runtime
   - use the `heim clear` command to remove all data from cache

   {" "}

   <br />

   :::caution[heim clear]
   You will need to redeploy the application the next time you want to run it, if you use the `heim clear` command.
   :::
