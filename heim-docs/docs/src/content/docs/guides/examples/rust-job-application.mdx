            #### component.toml

            The [component.toml](./../../../application/config/componenttoml/) file, defines a component, and one or more components make up an application in Heim.

            <br/>

            `component.toml`
            ```toml
            name = "my-job-trigger"
            version = "0.1.0"
            
            [build.dev]
            build = "cargo build --target=wasm32-wasip2"
            wasm_path = "target/wasm32-wasip2/debug/my_job_trigger.wasm"
            
            [build.prod]
            build = "cargo build --target=wasm32-wasip2 --release"
            wasm_path = "target/wasm32-wasip2/release/my_job_trigger.wasm"
            ```
            _a simple component.toml, with explanatory comments_

            <LinkCard title="Read more about component.toml" href="./../../../application/config/componenttoml/" />

            #### application.toml

            The [application.toml](./../../../application/config/applicationtoml/) is what defines an application. A application.toml is required for deploying an application to Heim.

            <br/>

            `application.toml`
            ```toml
            name = "my-job-trigger"
            component_target = 'my-job-trigger:0.1.0'
            
            [[trigger]]
            type = 'job'
            timeout = 20
            allow_parallel = true
            
            [capabilities]
            allowed-outgoing-url = ["https://dummyjson.com/*"]
            ```
            _a simple application.toml, with explanatory comments_

            `lib.rs`
            ```rust
            use clap::{Parser, Subcommand};
            use std::{env, error::Error};
            
            #[derive(Parser, Debug)]
            #[command(name = "Clap args", about = "A Heim job demo, for arguments")]
            struct Cli {
                #[command(subcommand)]
                command: Commands,
            }
            
            #[derive(Subcommand, Debug)]
            enum Commands {
                DummyJson {
                    #[arg(short, long, default_value = "recipe")]
                    path: String,
                },
            }
            
            const DUMMY_JSON_BASE_URL: &str = "https://dummyjson.com";
            
            fn main() {
                if let Err(err) = runtime::run() {
                    eprintln!("{err}");
                    std::process::exit(1);
                }
            }
            
            #[cfg(target_os = "wasi")]
            mod runtime {
                use std::error::Error;
                #[wstd::main]
                pub async fn run() -> Result<(), Box<dyn Error>> {
                    wstd::runtime::block_on(async move { super::async_main().await })
                }
            }
            
            #[cfg(not(target_os = "wasi"))]
            mod runtime {
                use std::error::Error;
            
                pub async fn run() -> Result<(), Box<dyn Error>> {
                    let rt = tokio::runtime::Runtime::new().unwrap();
            
                    rt.block_on(async move { super::async_main().await })
                }
            }
            
            async fn async_main() -> Result<(), Box<dyn Error>> {
                let cli = Cli::try_parse_from(env::args())?;
            
                match cli.command {
                    Commands::DummyJson { path } => {
                        let url_with_path = format!("{DUMMY_JSON_BASE_URL}/{}", path);
                        let contents = http_get_request(&url_with_path).await?;
                        println!("We performed a GET operation against {url_with_path}");
                        println!("Response: {contents}");
                    }
                }
            
                Ok(())
            }
            
            pub async fn http_get_request(url: &str) -> Result<String, Box<dyn Error>> {
                // Wasi
                #[cfg(target_os = "wasi")]
                {
                    use wstd::http::{Body, Client, Request};
                    let request = Request::get(url).body(Body::empty())?;
                    let response = Client::new().send(request).await?;
                    let mut body = response.into_body();
                    let contents = body.str_contents().await?;
                    Ok(contents.to_string())
                }
                // Not Wasi
                #[cfg(not(target_os = "wasi"))]
                {
                    use reqwest::Client;
                    let client = Client::builder().build()?;
                    let response = client.get(url).send().await?;
                    let text = response.text().await?;
                    Ok(text)
                }
            }

            ```

## Summary

<br />

1. - run `heim new`
   - select the rust-http `template`
   - choose `Max-execution time`
   - choose `allow_parallel`
   - set application `name`

   {" "}

   <br />
   <br />

2. - open the created application folder/project in your text editor

   _Take note of the configuration files:_

   :::note[component.toml]
   component.toml defines a component.

   An application is made up by one or more components.
   :::

   :::note[application.toml]
   application.toml defines an application.
   :::

   <br />

   <br />

3. - open `main.rs` in the `src` folder:

   :::note[main.rs]
   main.rs contains boilerplate source code for your application.
   :::

   {" "}

   <br />

   - run `heim start` to start the Heim runtime

   :::note[Heim portal]
   The Heim portal comes included with a Heim installation.

   Once the Heim runtime is started, the Heim portal is available on [http://127.0.0.1:3000/heim/portal/](http://127.0.0.1:3000/heim/portal/).
   :::

   {" "}

   <br />

   - run `heim deploy` to build and deploy the application.

   :::note[heim deploy --release]
   The `heim deploy --release` creates a _release_ build.

   Just type `heim deploy` to create a _development_ build.
   :::

   {" "}

   <br />

   - run `heim deploy --cloud` to deploy the application to the cloud instance.

   {" "}

   <br />

   :::tip[heim runtime]
   The Heim runtime will print out relevant information about the application.
   :::

   <br />

   <br />

4. - Trigger an execution of the application in Heim-Portal

   {" "}

   <br />

   {" "}

   <br />

5. - use the `Ctrl+C` shortcut to shut down the Heim runtime
   - use the `heim clear` command to remove all data from cache

   {" "}

   <br />

   :::caution[heim clear]
   You will need to redeploy the application the next time you want to run it, if you use the `heim clear` command.
   :::
